Sidenav/Overlay

Wrap sidenavs in a div with an ID of your choice. Sidenavs should contain:

  1. An open trigger with class="trigger"
  2. A close trigger with class="close-trigger"
  3. A sidenav div with class="sidenav"

In case you want to define external triggers, you can use the Sidenav.open() and Sidenav.close() methods

To use sidenavs, simply ensure that the Sidenav component is included in the script:

<​script src="/path/to/sidenav.js"><​script>

Then initialize the sidenav with:

const sidenav = new Sidenav('sidenav-id')

To programatically open or close a sidenav, simply do:

sidenav.open(); // opens the sidenav
sidenav.close(); // closes the sidenav

You can also create Overlays (no animation) with the following code:

const overlay = new Overlay('overlay-id')

Overlays contain the same methods that Sidenavs have

You can have as many sidenavs on a page as you want as long as their wrapping div ID is unique.

Responsive helpers

Responsive breakpoints are defined as follows:

  1. Phone: Below 600px
  2. Tablet portrait: Above 600px
  3. Tablet landscape: Above 900px
  4. Desktop: Above 1200px
  5. Large desktop: Above 1800px

If you are wondering why these breakpoints are chosen instead of the "standard" 768px, 1024px points, this is the article these breakpoints are based on: The 100% correct way to do css breakpoints

Using these breakpoints, a total of 8 helper classes are created:

Class Meaning
.only-phone Visible at or below 599px
.below-tablet-landscape Visible at or below 899px
.below-desktop Visible at or below 1199px
.below-large-desktop Visible at or below 1799px
.above-tablet-portrait Visible at or above 600px
.above-tablet-landscape Visible at or above 900px
.above-desktop Visible at or above 1200px
.above-desktop-large Visible at or above 1800px
